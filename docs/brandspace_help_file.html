A. HTML Structure - top

This template consists of 15 HTML page layout templates. Each page have it's own set of features, some shared with others and some unique to that page. When customising this template to suit your needs, you are free to mix and match these layout template is what ever way you require. Now we will take each page template in turn and describe the features of that specific page.

B. CSS Files and Structure - top

  • Stylesheets

    This template uses 5 CSS stylesheets in total. These are broken down into 3 types:

    • Helper Stylesheets.
    • Main Styles.
    • Additional Skin Styles.

  • Helper Stylesheets

    These css files consist of css frameworks to help with the creation of a blank canvas on which to build the main style information for the template. The achieve this by resetting the default styles of each browser and setting some new base styles for all items inlcusing typography. Another function of these styles is to provide a grid framework which makes it easier to achieve consistant page layouts acorss the whole template. The helper css files used in this template are as follows:

    • Global Browser Reset.
    • Basic Typographic Styles.
    • 960 Grid Framework.

  • Main Styles

    This is the main css file for the template. The files is strictured in the following way:

    • Helper CSS3 classes.
    • General template-wide styles.
    • Page specific styles - each page in turn.
    • Conditional styles for Internet Explorer 7.
    • Conditional styles for Internet Explorer 8.
    • Conditional styles for Internet Explorer 9.
    • Styles for PrettyPhoto lightbox plugin.

    Each section of this stylesheet is marked with comments to help you find it, should you require to change anything.

  • Skin Stylesheets

    This template includes a dark skin in addition to the default light skin. The styles for the dark skin have been created in a separate stylesheet named "dark.css". This stylesheet only contains color, border and background image information and has nothing to do with any layout styles, and therefore this stylesheet must work along side the default stylesheet (See above). To activate the dark skin simply uncomment the line of code in the head section of each page's html file. Ensure this line is lower in the code than the line linking the main stylesheet. This line will look like this, just activate the line and the dark skin is switched on:

    	<!-- color stylesheet -->
	<!--<link rel="stylesheet" href="css/dark.css" />-->

    To creake a new skin, all you have to so is copy the "dark.css" to a new file. Change the required style information, and insert your new skin stylesheet like the "dark.css" as shown above. The dark stylesheet has been structured similarily to the main styesheet, as follwos:

    • Template-wide dark styles.
    • Page specific dark styles - each page at a time.
    • Conditional dark styles for Internet Explorer 7.

  • Editing Stylesheets

    In general, items are labelled semantically, so whether you are looking for the newsletter signup form, the testimonial section or one of the banner elements, simply look for this word in the CSS selectors and you will be in the right place. The CSS file follows a top to bottom sequence which is similar to the html. This measn the footer styles will be below the main content styles which will in turn be below the header styles, and each will be marked with comments.

    It is alos a good idea when attempting to edit css styles, to use some kind of browser based source code inspector. The Google Chrome and Opera Browsers have once of these built in, while Mozilla Firefox requires ayou to download and install the "Firebug" add on, available here. To activate these inspectors simple right-click on an element who's style and mark-up information you would like to inspect, and select "Inspect Element" from the context menu. Amongst other things, this will tell you what selector you will need to edit in the css file in order to change the element's appearance.

  • Homepage Banner Slideshow

    Working Mode

    The slider component can work in 2 different modes.

    • Ken Burns Mode - image is zoomed out/in and/or panned to defined direction according to each slide settings when slide changes, a crossfade transition is used.
    • Swipe Mode - a custom "swipe" transition will be applied when a new slide is loaded.

    The desired working mode can be chosen by using the "data-animate" attribute of the Banner's containing div. See below.

    Ken Burns Mode

    <div id="banner"  class="grid_16" data-animate="true" >

    Swipe Mode

    <div id="banner"  class="grid_16" >
    Info Panel

    The info panel's behaviour can be controlled with the "data-panel" attribute of the Banner's containing div. See below.

    <div id="banner"  class="grid_16" data-animate="true" data-panel="disabled">

    This "data-panel" attribute accepts any of the following values:

    • "hidden" - hidden by default
    • "disabled" - hidden by default and show button not available
    • "hideOnFirst" - visible by default, than auto hidden on first slide transition
    • "opened" - visible by default (you can change "opened" to any string you like)

    Per Slide Settings

    Each banner slide can have custom values to control keen burns, timings and transition type settings are defined using data-xxxx attributes, for example:

    <div class="active" data-delay="5" data-zoom="out" 
     data-align="random" data-pan="random" data-duration="15">

    The possible settings and their allowed values:

    • Delay Setting

      Accepts a number, which defines the number of seconds before auto-loading of the next slide.

    • Transition Settings
      • "swipe" - swipe transition
      • "flyBy" - flyBy transition
      • "fade" - crossfade transition
      • "whitefade" - fade to white

    • Ken Burns Settings
      • data-duration - kb transition duration in seconds (the higher the value, the slower the movement will be)
      • data-zoom - kb zoom mode. Accepted values are: "out", "in" or "random"
      • data-align - kb image alignment, format is "vertical, horizontal"

        • vertical values: "top", "center", "bottom"

        horiz values: "left", "center", "right"

        example: "top,left" or "bottom,right" or "random"

      • data-pan - kb movement direction (from "data-align" to "data-pan")

        • vertical values: "top", "center", "bottom"

        horiz values: "left", "center", "right"

        example: "top,left" or "bottom,right" or "random"

    Lazy Loading

    The Banner has been build to include Lazy Loading. Lazy loading means that each images in your slideshow will only be loaded when is about to be displayed. If your slideshow has a lot of images, you can activate lazy loading by setting image "src" attribute to a blank, small png (see download index.heml file for example) while using the "data-src" attribute to define real path of the image. See below.

    <img data-src="img/home/banner_13.jpg" src="img/icons/blank.png" alt="Banner Image 3" />
    Video Slides

    This Banner has been designed to support video slides. Videos from Vimeo or YouTube may be shown. The slideshow will use flash to load the embedded video player, if flash is not available the banner will automatically redirect user to youtube or vimeo page. On iDevices this link should automatically open the default YouTube application shipped with Apple's iOS, while in the case of Vimeo the user will be taken to the HTML5 fallback version of Vimeo's video player. To embed a video from one of these sites,see the following example:

    <a class="video hd autoplay loop" href="http://vimeo.com/7809605">
	<img src="img/home/banner_12.jpg" alt="video cover image" />
</a>
    • href - full Vimeo or YouTube single video url
    • class
      • video (required for loading a video into slideshow window)
      • hd (use hd resource)
      • autoplay (auto start playback)
      • loop (loop when video ends)

    Normal Resource Links

    If class="video" is not defined, any resources will be loaded like a normal link

D. CSS & Javascript Compressor - top

Included in package download is a compressor for both js and css files When using the compressor, all external css and js files are merged in a single compressed files. This has 2 advantages - both of which mean your page will load faster.

  • A single request to server is used instead of multiple requests.
  • The single file is gzip compressed resulting in very small size for the css and js.

The compressor requires php so it can only be enabled if supported by the hosting plan. To enable the compressor for a given page, edit the head section and remove all script and link tags used to load external js or css and instead, add a single request for js and css. The code required for this is already located in the head section of each HTML file of this template, and look like this:

<!--CSS Compressor-->
<!--Replace all css import statements with 1 of these options - see help file for more info.-->
<!-- option 1 -->
<!--<link rel="stylesheet" href="packs/jscsscomp.php?q=pack.css" />-->
<!-- option 2 -->
<!--<link rel="stylesheet" href="packs/pack.css" />-->
<!--end css compressor-->
<!--Js Compressor-->
<!--Replace all js import statements with 1 of these options - see help file for more info.-->
<!-- option 1 -->
<!--<script type="text/javascript" src="packs/jscsscomp.php?q=pack.js"></script>-->
<!-- option 2 -->
<!--<script type="text/javascript" src="packs/pack.js"></script>-->
<!--end js compressor-->

As you can see, the compressor is passed a value (pack.js for js and pack.css for css). This value refers to a file which is located in the "packs" folder, has name of "pack.css" (compressor value) + ".files" (extension) and contains a list of each file that will be merged in the single compressed js or css. Here you can see an example of pack.css.files:

minified/reset.css
minified/960.css
minified/text.css
minified/style.css
minified/prettyPhoto.css

The "minified" folder has minified version of all js and css used for this template."Minified" basically means that all comments, unnecessary line breaks and spaces have been stripped from the original file, leading to a more optimized version. If you need to edit one of the source css or javascript files, you can minify it again use an online services like http://refresh-sf.com/yui/. Once that has been done, you can save the minified file into the "minified" folder.

Caching Compressed Files

Compressing files can be quite a resource intensive operation so the compressor has the ability to save compressed files in the "packs/cache" directory instead of performing the same operation again and again for this to work, you need to make sure that "cache" folder is writable by php.

Pretty Urls

If your hosting plan has apache mod rewrite support, you can enable the rules in .htaccess file which has been included in the download package. To enable rules, simply uncomment them. Enabling the following rule will allow you to use the more pretty compressor import statements, shown below, in the head of all HTML files:

RewriteRule   ^packs/(\w+)\.(css|js)	packs/jscsscomp.php?q=$1.$2	[L,NC]

Pretty

<script type="text/javascript" src="packs/pack.js"></script>
<link rel="stylesheet" href="packs/pack.css" />

Not pretty

<script type="text/javascript" src="packs/jscsscomp.php?q=pack.js"></script>
<link rel="stylesheet" href="packs/jscsscomp.php?q=pack.css" />

<!-- font replacement -->

G. Newsletter Form Usage & Customisation - top

This template contains a newsletter sign up form. This form functions by sending an email from the email address, which has been entered into the newsletter signup input field, to an address specified by the site owner, in the "newsletter.php" file. This PHP file can be found in the php folder inside the download package.

The subject and content of this email may be altered in this php file, to suit the requirements of your newsletter provider. The locations of these alterations are marked with comments in the PHP file and are shown here below.

$subscribe = "youremail@yourdomain.com";  // insert your subscribe email address here
$subject = "subscribed to the newsletter"; //subscribe email subject line & content
	echo "THANK YOU FOR SUBSCRIBING"; //message appearing in form

The default message appearing in this newsletter signup field must be entered into the HTML mark-up of each page in which it appears. The default message must be entered twice. Once in the "value" attribute of the input element and a second time into the "data-default" attribute of the newsletter container "div". You can see both of these instances in the following code:

<!--newsletter signup form-->
<div id="newsletter" data-default="youremail@yourdomain.com" data-subscribed="Thank you for subscribing.">
	<form action="#" method="POST">
	    <input type="text" name="email" value="youremail@yourdomain.com" />
	    <input type="submit" value="submit" />
	</form>
	<h2>Sign Up to Our Newsletter!</h2>
</div>

When an email has been successfully sent from this form a custom message is displayed to the user. This message can be altered also via the "data-subscribed" attribute of the newsletter container "div", as seen in the above code.

This template contains a combined contact and get-a-quote form. The get-a-quote section of the form is optional and maybe removed from the page by adding a class of "noquote" to the "form" tag. This is illustrated in the following code snippets:

Quote form deactivated:

In the case of an error being detected.
